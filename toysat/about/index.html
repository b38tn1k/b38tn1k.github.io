Toy Sat Help

Toy Sat is a naive physics model of a fully actuated 2D cube satellite. All motion is calculated based on physical properties:

  • F = ma
  • M = Fd
  • etc...

The thrusters output a constant force each video frame. Every frame a thruster is active, the fuel level is reduced, reducing the mass of the satellite - the satellite accelerates way faster when it is low on fuel! You can see the fuel / mass level on the RHS of the screen

You can control the individual thrusters by clicking or pressing on the buttons, or use the keyboard. The grey buttons combine typical thruster combinations for understandable motion. You can use the keyboard arrow keys for direction and 'a' and 'z' to set rotation using these thruster combinations.

The round red button 'q' triggers an open loop control function that calculates thruster sequences and durations that will bring the stationary to a stop before navigating it back to the middle of the screen. The control sequence is:

  • stop any spin
  • stop the trajectory
  • rotate the satellite to orient with the screen (start and stop burns)
  • fly the satellite back to the center of the screen (start and stop burns)
  • enter HOLD POSITION mode until a new manual input is made

You will notice some inefficiencies in this approach - if the satelite is already spinning, why stop it spinning at an arbitraty angle only to start it again to reach the desired angle? Momentum is a useful tool, I don't use it here. 

The physics will also fall apart at high speed, acceleration, or rotation values. This is running in a webbrowser after all. You can break it nicely by getting the satellite up to speed and in a spin, then hitting the red button.
